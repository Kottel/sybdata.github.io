How to compile and install Linux Kernel 4.x from source code(*Debian base)

Um zu kompilieren und den neuesten Linux-Kernel aus dem Quellcode zu installieren, gehen Sie wie folgt vor:

  1. Holen Sie sich den neuesten Kernel von kernel.org
  2. Überprüfen Sie den Linux-Kernel-Tartball mit pgp
  3. Entpacken Sie das Kernel-Archiv
  4. Kopieren Sie die vorhandene Linux-Kernel-Konfigurationsdatei
  5. Linux-Kernel kompilieren und erstellen 4.20.10
  6. Linux Kernel und Module (Treiber) installieren
  7. Aktualisieren Sie die Grub-Konfiguration
  8. Starten Sie das System neu

*Achten Sie darauf, dass genügend Speicherplatz auf der Festplatte frei ist: < ~50 GB

Schritt 1. Holen Sie sich den neuesten Linux kernel source code

Besuchen Sie die offizielle Projektseite und laden Sie den neuesten Quellcode herunter. 

https://www.kernel.org/
https://cdn.kernel.org/pub/linux/kernel/v4.x/   # Index Katalog

Stable Kernel: 4.20.10

Verwenden Sie den Befehl wget, um Linux-Kernel-Quellcode herunterzuladen:

cd /usr/src
wget https://cdn.kernel.org/pub/linux/kernel/v4.x/linux-4.20.10.tar.xz

Schritt 2. Entpacken Sie die tar.xz-Datei

xz -d -v linux-4.20.10.tar.xz

Überprüfen Sie den Linux-Kernel-Tartball mit pgp

Besorgen Sie sich zuerst die PGP-Signatur für linux-4.20.10.tar:

wget https://cdn.kernel.org/pub/linux/kernel/v4.x/linux-4.20.10.tar.sign

Wenn Sie keine Ausgabe der „BAD-Signatur“ vom Befehl erhalten, entpacken Sie das Linux-Kernel-Tarball mit dem Befehl tar: 

tar xf linux-4.20.10.tar

Schritt 3. Konfigurieren Sie die Linux-Kernel-Funktionen und -Module

Bevor Sie mit der Erstellung des Kernels beginnen, müssen Sie die Linux-Kernel-Funktionen konfigurieren. Sie müssen außerdem angeben, welche Kernelmodule (Treiber) für Ihr System erforderlich sind. Die Aufgabe kann für einen neuen Benutzer überwältigend sein. Ich empfehle Ihnen, die vorhandene Konfigurationsdatei mit dem Befehl cp zu kopieren:

cd linux-4.20.10
cp -v /boot/config-$(uname -r) .config

Ausgabe:

'/boot/config-4.9.0-8-amd64' -> '.config'

Schritt 4. Installieren Sie die erforderlichen Compiler und andere Tools

Sie müssen Entwicklungstools wie GCC-Compiler und verwandte Tools installiert haben, um den Linux-Kernel zu kompilieren.

apt-get install build-essential libncurses-dev bison flex libssl-dev libelf-dev bc bzip2 make chrpath gawk texinfo libsdl1.2-dev whiptail diffstat cpio

Schritt 5. Den Kernel konfigurieren

Jetzt können Sie die Kernelkonfiguration starten, indem Sie einen der folgenden Befehle in das Quellcode-Verzeichnis eingeben:

  • $ make menuconfig – Text based color menus, radiolists & dialogs. This option also useful on remote server if you wanna compile kernel remotely.
  • $ make xconfig – X windows (Qt) based configuration tool, works best under KDE desktop
  • $ make gconfig – X windows (Gtk) based configuration tool, works best under Gnome Dekstop.
For example, run make menuconfig command launches following screen:
make menuconfig

Bitte beachten Sie, dass ‚make menuconfig‘ optional ist. Ich habe es hier nur zu Demonstrationszwecken verwendet. Sie können bestimmte Funktionen oder Kernel-Treiber mit dieser Option aktivieren oder deaktivieren.

Schritt 5. So kompilieren Sie einen Linux-Kernel

Zurück zum Schritt 3, meist stammt diese Konfiguration von einem älteren Kernel als dem, den zu bauen man im Begriff ist. In diesem Fall fehlen der Konfiguration eventuell Optionen zu neueren Funktionen. Um dies zu vermeiden, erstellt man eine vollständige Konfigurationsdatei. Der folgende Befehl liest die vorhandenen Einstellungen und fragt die Optionen für neue, nicht definierte Funktionen ab. Ist man sich unsicher, was man auswählen sollte, kann man auf die Vorgaben vertrauen. Diese sind immer die Standard-Option und können per »Enter« ausgewählt werden.

Ist man des minutenlangen Betätigens der »Enter«-Taste überdrüssig, so kann man auch den Befehl yes zu Hilfe nehmen. Dieser simuliert das Drücken der »Enter«-Taste, bis make oldconfig durch alle Optionen gelaufen ist:

yes "" | make oldconfig
yes "" | make oldconfig  # verzichtet dagegen ganz auf irgendwelche Abfragen und übernimmt stattdessen die von den jeweiligen Entwicklern vorgesehenen Standardeinstellungen in die Konfigurationsdatei
*Weiteren Targets zum Konfigurieren:
config: # alle Einstellungen werden in der Kommandozeile abgefragt 
defconfig: # Kernel-Standardkonfiguration entsprechend der jeweiligen Architektur
oldconfig: # vorliegende Konfiguration wird übernommen, neue Funktionen/Treiber werden in der Kommandozeile interaktiv abgefragt 
localmodconfig: # vorliegende Konfiguration wird übernommen, nicht aktive Module werden deaktiviert 
localyesconfig: # wie localmodconfig, aber die Module werden in den Kernel fest eingebunden 
allnoconfig: # neue Konfiguration, alle Abfragen werden mit "Nein" beantwortet 
allyesconfig: # neue Konfiguration, alle Abfragen werden mit "Ja" beantwortet 
allmodconfig: # neue Konfiguration, es werden Treiber als Modul konfiguriert 
alldefconfig: # neue Konfiguration, alle Komponenten mit Standard-Optionen 
randconfig: # neue Konfiguration, Optionen werden zufällig gesetzt 
menuconfig: # Terminalbasierte Oberfläche, basierend auf ncurses 
nconfig: # Terminalbasierte Oberfläche, basierend auf ncurses 
xconfig: # grafische Oberfläche, basierend auf Qt 
gconfig: # grafische Oberfläche, basierend auf GTK+

Der folgende Befehl baut den oben konfigurierten Kernel und die zugehörigen Module zusammen

Und um die Kompilierzeit zu beschleunigen, übergeben Sie -j wie folgt:

## 4 Core / Thread verwenden ## 

make -j4 bzImage modules

Kompilieren und der Aufbau des Linux-Kernels eine erhebliche Menge an Zeit gehen nehmen. Die Erstellungszeit hängt von den Ressourcen Ihres Systems ab, wie zum Beispiel dem verfügbaren CPU-Kern und der aktuellen Systemlast. Also etwas Geduld.

Install the Linux kernel und die Module

make modules_install install

Es werden drei Dateien im / boot-Verzeichnis installiert und die Kernel-Grub-Konfigurationsdatei geändert:

  • initramfs-4.20.10.img
  • System.map-4.20.10
  • vmlinuz-4.20.10

Schritt 6. Aktualisieren Sie die Grub-Konfiguration

Die folgenden Befehle sind optional, da make install alles für Sie tut, hier jedoch nur aus historischen Gründen enthalten ist:

update-initramfs -c -k 4.20.10
update-grub

Starten Sie den Linux-Computer neu und booten Sie in Ihrem neuen Kernel:

reboot

Überprüfen Sie nach dem Neustart die neue Linux-Kernel-Version:

uname -mrs
